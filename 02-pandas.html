פרק 2 · Pandas

DataFrames
וניתוח נתונים

Pandas הוא כלי העבודה היומיומי של data scientist. groupby, merge, apply — חייבים להכיר.

Pandas בנוי על NumPy, אבל מתמחה בנתונים טבלאיים (כמו אקסל). במבחן כמעט תמיד יש שאלה אחת על groupby/merge, וגם שאלה על קריאת קוד עם apply או lambda. הפרק הזה מכסה את מה שצריך לדעת.

שני האובייקטים המרכזיים

📘Series ו־DataFrame

Series — עמודה בודדת עם אינדקס

כל Series יש index (לא חייב להיות מספרים) ו־values:

series.py
import pandas as pd

# יצירה מ-dict
s = pd.Series({'a': 1, 'b': 2, 'c': 3})
# a    1
# b    2
# c    3

# גישה
s['a']        # 1
s.index       # Index(['a', 'b', 'c'])
s.values      # array([1, 2, 3])

DataFrame — טבלה דו־מימדית

אוסף של Series עם אותו index. כל עמודה יכולה להיות מטיפוס שונה.

dataframe.py
df = pd.DataFrame({
    'name':  ['Alice', 'Bob', 'Carol'],
    'age':   [25, 30, 35],
    'city':  ['TLV', 'NY', 'TLV']
})

# גישה לעמודה
df['age']          # Series
df.age             # אותו דבר (אם השם הוא identifier חוקי)

# גישה לשורה - שני אופנים
df.loc[0]          # לפי label
df.iloc[0]         # לפי position (תמיד 0, 1, 2...)

# סינון בוליאני
df[df['age'] > 28]
# Bob, Carol

# בחירת עמודות מרובות
df[['name', 'city']]

GroupBy — האופציה הכי שאלת מבחן

📘איך עובד groupby

groupby מחלק את הנתונים לקבוצות לפי ערכי עמודה אחת או יותר, ואז מבצע אגרגציה על כל קבוצה.

תחביר בסיסי

groupby_basic.py
df = pd.DataFrame({
    'team': ['A', 'A', 'B', 'B', 'B'],
    'score': [10, 20, 30, 40, 50]
})

# סכום ציונים פר team
df.groupby('team')['score'].sum()
# team
# A    30
# B   120

# כמה דרכים שונות לעשות את אותו דבר
df.groupby('team').sum()                 # על כל העמודות המספריות
df.groupby('team')['score'].sum()        # רק על score
df.groupby('team').agg({'score': 'sum'}) # agg מאפשר יותר גמישות
טעות נפוצה: ההבדל בין:
✗ df.groupby(['a']['b']) — שגיאה!
✓ df.groupby(['a', 'b']) — לעבד לפי שתי עמודות (list בתוך groupby).

groupby לפי שתי עמודות

groupby_two.py
products = pd.DataFrame({
    'region':  ['r001', 'r001', 'r002', 'r001'],
    'product': [444, 555, 444, 444],
    'sales':   [100, 50, 200, 150]
})

# סכום sales לכל (region, product)
products.groupby(['region', 'product'])['sales'].sum()
# region  product
# r001    444        250    ← 100+150
# r001    555         50
# r002    444        200

# שימוש ב־as_index=False — מקבלים DataFrame, לא Series
result = products.groupby(['region', 'product'], as_index=False)['sales'].sum()

Merge — חיבור בין DataFrames

📘ארבעת סוגי ה־join

נניח שיש לנו שני DataFrames:

  • df1 (employees): name, team
  • df2 (team_info): team, established
סוגתוצאה
innerרק שורות שמופיעות בשני ה־DataFrames (חיתוך).
outerכל השורות, NaN חסרים (איחוד).
leftכל השורות של df1, NaN אם אין התאמה ב־df2.
rightכל השורות של df2, NaN אם אין התאמה ב־df1.

תחביר

merge_examples.py
pd.merge(df1, df2, how='inner')                  # ברירת מחדל
pd.merge(df1, df2, how='outer')                  # כל השורות
pd.merge(df1, df2, how='left', on='team')        # mapping לפי team
pd.merge(df1, df2, how='right', left_on='id', right_on='emp_id')
טריק לזיכרון: חשוב כמו במתמטיקה — inner = ∩, outer = ∪.

Apply ו־Lambda

📘פעולות מותאמות אישית על שורות/עמודות

אם הפעולה שאתה רוצה לעשות לא קיימת כפונקציה מובנית, השתמש ב־apply עם lambda.

apply_lambda.py
df = pd.DataFrame({'name': ['Alice', 'Bob'], 'age': [25, 30]})

# apply על עמודה - lambda מקבל איבר אחד בכל פעם
df['age_in_5_years'] = df['age'].apply(lambda x: x + 5)
# 30, 35

# apply על כל הDataFrame עם axis=1 - lambda מקבל שורה
df['summary'] = df.apply(lambda row: f"{row['name']}, {row['age']}", axis=1)
# Alice, 25
# Bob, 30

# דוגמה נפוצה מהמבחן: עיבוד תאריך
df['date'] = ['12/05/2002', '17/11/2004']
df['year'] = df['date'].apply(lambda d: int(d.split('/')[2]))
# 2002, 2004

df['day'] = df['date'].apply(lambda d: int(d.split('/')[0]))
# 12, 17
טריק: lambda x: x + 1 זה בעצם פונקציה אנונימית שמחזירה x+1. השם ״לאמבדה״ בא ממתמטיקה.

תבניות שמופיעות במבחן

💡שלוש תבניות חוזרות

1. מציאת ערכים מעל הממוצע (פר־קבוצה)

״הראה את התלמידים שהציון שלהם מעל הממוצע של הכיתה שלהם״.

pattern1.py
# 1. חשב ממוצע פר־כיתה
tmp = students.groupby('class', as_index=False)['grade'].mean()
tmp.rename(columns={'grade': 'avg_grade'}, inplace=True)

# 2. merge עם הטבלה המקורית
merged = students.merge(tmp, on='class')

# 3. סנן
result = merged[merged['grade'] >= merged['avg_grade']]

2. סינון לפי תנאי על תאריך

״הראה תלמידים שיום הלידה שלהם זוגי״.

pattern2.py
students['day'] = students['dob'].apply(lambda d: int(d.split('/')[0]))
result = students[students['day'] % 2 == 0]

3. חישוב על קבוצה עם count + sum

שילוב של merge עם left ולחישוב סיכומים.

pattern3.py
pd.merge(df1, df2, how='left')['established'].count()  # סופר non-NaN
pd.merge(df1, df2, how='left')['established'].sum()    # סכום (NaN מתעלמים אוטומטית)

📝תרגיל 1 · practice2, שאלה 1
groupby נכון ולא נכון

נתון DataFrame בשם products עם עמודות: region, product, sales. הראה את סה״כ המוצרים שנמכרו לפי אזורים. למשל עבור region='r001' ו־product=444 יודפס 250.

סמנו את כל התשובות הנכונות:

  1. products.groupby(['region']['product'])['sales'].sum()
  2. products.groupby(['region']['product']).sum(['sales'])
  3. products.groupby(['region', 'product'])['sales'].sum()
  4. for key, df in products.groupby(['region', 'product']): print(key, df['sales'].sum())
  • 1. groupby(['region']['product']). לא נכון. ['region']['product'] זאת תחביר שגוי. נסיון לעשות subscript על list של מחרוזת אחת.
  • 2. groupby(['region']['product']).sum(['sales']). לא נכון. אותה בעיה כמו 1 בקריאה ל־groupby.
  • 3. groupby(['region', 'product'])['sales'].sum(). נכון. רשימה של עמודות לתוך groupby, אז בחירת עמודה, אז .sum().
  • 4. Loop עם groupby. נכון. iteration על groupby נותן (key, sub_df). אחר־כך מדפיסים את הסכום של ה־sub_df. גם זה ידפיס את הסכום הנכון, רק בפורמט שונה.
נכונות: 3 ו־4.
📝תרגיל 2 · practice2, שאלה 2
merge — ארבעת סוגי join

נתונים שני DataFrames של עובדים:

df1 (3 שורות): name + team. Moshe-development, Adam-research, Emily-HR.

df2 (3 שורות): team + established. development-2008, research-2013, finance-2010.

תאר את הפלט של כל אחת מהפקודות:

  1. pd.merge(df1, df2, how='inner')
  2. pd.merge(df1, df2, how='outer')
  3. pd.merge(df1, df2, how='left')['established'].count()
  4. pd.merge(df1, df2, how='left')['established'].sum()
סעיף 1 · inner join

רק שורות שמופיעות בשני ה־DataFrames (team גם כאן וגם כאן):

name team established 0 Moshe development 2008 1 Adam research 2013

(Emily ב־HR לא מופיעה ב־df2. finance לא מופיע ב־df1.)

סעיף 2 · outer join

כל השורות מאיחוד:

name team established 0 Moshe development 2008 1 Adam research 2013 2 Emily HR NaN 3 NaN finance 2010
סעיף 3 · left join, count על established

left = כל השורות של df1 (Moshe, Adam, Emily). אחרי merge:

name team established 0 Moshe development 2008 1 Adam research 2013 2 Emily HR NaN

.count() סופר non-NaN. אז: 2.

סעיף 4 · left join, sum על established

אותו merge, אבל .sum() במקום .count():

$$2008 + 2013 = 4021$$
1) inner = 2 שורות (Moshe, Adam). 2) outer = 4 שורות (כולל NaN). 3) count = 2. 4) sum = 4021.
📝תרגיל 3 · practice2, שאלה 3
apply + groupby + merge

נתונה טבלת תלמידים עם עמודות: school_code, class, name, date_of_birth (פורמט DD/MM/YYYY), grade.

(ג) מה מודפס:

part_c.py
tmp = student_data.groupby('class', as_index=False)['grade'].mean()
tmp.rename(columns={'grade': 'tmp_col'}, inplace=True)
merged = student_data.merge(tmp, on='class')

result = merged[merged['grade'] >= merged['tmp_col']]
print(result)

(ד) תאר במילים מה הקוד עושה.

שלב 1 · חישוב ממוצע פר־כיתה

השורה הראשונה יוצרת DataFrame קטן עם הממוצע של כל כיתה: כיתה 4 = 79.60, כיתה 5 = 70.40, כיתה 6 = 88.70, כיתה 7 = 76.65.

שלב 2 · merge מחזיר עמודת tmp_col לכל תלמיד

אחרי merge(tmp, on='class'), כל תלמיד מקבל את הממוצע של כיתתו.

שלב 3 · סינון לפי ציון ≥ ממוצע כיתה

נשארים רק תלמידים שהציון שלהם מעל ממוצע הכיתה שלהם.

(ד) הקוד מדפיס את פרטי כל הסטודנטים שהציון שלהם גבוה או שווה לממוצע הכיתה שלהם.

סיכום מהיר לזיכרון

📌שמונה נקודות שאתה חייב לזכור
  1. Series = עמודה בודדת עם index. DataFrame = אוסף של Series.
  2. df.loc[i] — גישה לפי label. df.iloc[i] — לפי position.
  3. groupby(['col1', 'col2'])['col3'].sum() — list של עמודות לקיבוץ, ואז בחירת עמודה.
  4. as_index=False מחזיר DataFrame במקום Series עם index מורכב.
  5. merge: inner (חיתוך), outer (איחוד), left (שמירת df1), right (שמירת df2).
  6. apply + lambda מאפשרים פעולות מותאמות. apply(lambda x: ...).
  7. axis=1 ב־apply = לעבד שורה־שורה. axis=0 = עמודה־עמודה.
  8. .count() מתעלם מ־NaN. .sum() גם — לכן הוא לא משפיע.
