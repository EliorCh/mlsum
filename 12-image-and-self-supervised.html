פרק 12 · עיבוד תמונה

תמונות כ־Feature Vectors

איך מטפלים בתמונות עם הכלים שכבר למדנו. PCA על digits, autoencoders לתמונות, drift detection בתמונות.

אין הרצאה ייעודית על ״עיבוד תמונה״ בקורס — אבל הנושא מובלע בתוך מה שכבר למדנו. הסוד: תמונה דיגיטלית היא פשוט וקטור של מספרים (פיקסלים), וכל הכלים שלנו (PCA, autoencoder, logistic regression) עובדים עליה בדיוק כמו על כל data אחר. בפרק הזה נראה את ה־framework, נעבור על דוגמת ה־digits הקלאסית, וניישם autoencoders ו־drift detection בהקשר תמונות.

הקונספט המרכזי: תמונה = וקטור

📘איך תמונה נראית למחשב

תמונה דיגיטלית בגווני אפור היא מטריצה של פיקסלים. כל פיקסל הוא מספר בין 0 ל־255 (או 0 ל־1 אחרי נרמול), שמייצג את עוצמת האור.

תמונה של ספרה בגודל 8×8 פיקסלים:

  • סך הכל 64 פיקסלים.
  • כל פיקסל = ערך מספרי.
  • ״משטחים״ את המטריצה לוקטור: $\mathbf{x} = [x_1, x_2, \ldots, x_{64}]$.
תמונת ספרה דיגיטלית (8×8) מומרת לוקטור של 64 מספרים. עכשיו אפשר להפעיל עליה כל מודל ML שיודע לעבוד על וקטורים.
נקודת המפתח: אחרי ה־flatten, התמונה היא וקטור 64-מימדי. עכשיו אותם כלים שלמדנו (PCA, KNN, רגרסיה לוגיסטית, רשתות נוירונים, autoencoders) עובדים עליה.

הדוגמה הקלאסית: סיווג ספרות (Digits)

📘נתוני sklearn.datasets.load_digits

זאת הדוגמה הקאנונית של עיבוד תמונה במבוא ל־ML, מהpca_classification_digits_example.html שבחומר הקורס.

נתוני הבעיה

  • 1,797 תמונות של ספרות בכתב יד (0-9).
  • כל תמונה 8×8 פיקסלים בגווני אפור.
  • אחרי flatten: 64 features.
  • 10 מחלקות (ספרה 0, ספרה 1, ..., ספרה 9).
  • חלוקה: 1,347 train / 450 test (75/25).

למה זאת בעיית benchmark מצוינת?

  • מספיק קטנה לרוץ מהר.
  • מספיק מאתגרת — 10 מחלקות, כתב יד שונה.
  • מאפשרת לבחון השפעת PCA וטכניקות אחרות באופן ברור.

שלב 1: סיווג ישיר על 64 הפיקסלים

🎯Logistic Regression על raw pixels

נריץ logistic regression ישירות על הוקטור של 64 פיקסלים (אחרי נרמול עם StandardScaler):

classify_before_pca.py
from sklearn.datasets import load_digits
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import accuracy_score

digits = load_digits()
X, y = digits.data, digits.target   # X: (1797, 64), y: (1797,)

X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.25, stratify=y, random_state=42
)

# Scaling - required before logistic regression
scaler = StandardScaler()
X_train_s = scaler.fit_transform(X_train)
X_test_s  = scaler.transform(X_test)

# Classify
clf = LogisticRegression(max_iter=2000)
clf.fit(X_train_s, y_train)
y_pred = clf.predict(X_test_s)

print(f'Accuracy: {accuracy_score(y_test, y_pred):.4f}')
# Accuracy: 0.9778

תוצאה: 97.78% דיוק. נחמד מאוד.

אבל יש שתי בעיות פוטנציאליות:

  • Multi-collinearity בין פיקסלים — פיקסלים שכנים בתמונה כמעט תמיד מתואמים.
  • זמן אימון — 64 features זה לא הרבה, אבל בתמונה רגילה יש מיליוני פיקסלים.

שלב 2: הפעלת PCA

📘אסטרטגיה: מ־64 ל־k מימדים

אחרי שראינו בפרק 11 איך PCA עובד, נשתמש בו על נתוני ה־digits:

pca_on_digits.py
from sklearn.decomposition import PCA
import numpy as np

# Fit PCA on train set only (no data leakage!)
pca_full = PCA()
pca_full.fit(X_train_s)

# Compute cumulative explained variance
cumvar = np.cumsum(pca_full.explained_variance_ratio_)

# How many components to explain X% variance?
n_90 = np.searchsorted(cumvar, 0.90) + 1
n_95 = np.searchsorted(cumvar, 0.95) + 1

print(f'Components for 90% variance: {n_90}')   # 31
print(f'Components for 95% variance: {n_95}')   # 40
נתון מרכזי: אפשר להוריד את המימד מ־64 ל־31 תוך שמירה על 90% מהשונות, או ל־40 ל־95%. זה אומר שיש redundancy רבה בפיקסלים — לא כל הפיקסלים נושאים מידע בלתי תלוי.

שלב 3: סיווג אחרי PCA

🎯השוואת ביצועים

נריץ את אותו logistic regression, אבל אחרי הורדת מימד:

classify_after_pca.py
# PCA with 31 components (90% variance)
pca = PCA(n_components=31)
X_train_pca = pca.fit_transform(X_train_s)
X_test_pca  = pca.transform(X_test_s)

# Classify
clf2 = LogisticRegression(max_iter=2000)
clf2.fit(X_train_pca, y_train)
y_pred2 = clf2.predict(X_test_pca)

print(f'Accuracy: {accuracy_score(y_test, y_pred2):.4f}')
# Accuracy: 0.9489

טרייד־אוף: פחות מימדים = אימון מהיר יותר ופחות בעיות multi-collinearity, אבל איבוד דיוק. עם 2 רכיבים בלבד אפשר ויזואליזציה ב־2D, אבל ה־accuracy צונח ל־51%. עם 31 רכיבים — אובדן קטן (3% פחות).
Accuracy עולה בחדות בערך 10 הרכיבים הראשונים, אז מתייצב. אחרי 30 רכיבים, ההוספה כמעט לא תורמת.

Autoencoders לשחזור תמונות

📘למה autoencoders עובדים טוב על תמונות

כפי שראינו בפרק 9, autoencoder לומד לדחוס נתונים ולשחזר אותם. תמונות הן המקרה הקלאסי שבו זה עובד מצוין:

  • Redundancy גבוהה בתמונות — פיקסלים שכנים דומים. ה־encoder יכול לתפוס תבניות חוזרות.
  • Latent space דחוס יכול לייצג מאפיינים סמנטיים — ״עיגול״, ״קו אנכי״, ״קו אלכסוני״.
  • שחזור מתמונה דחוסה אפשרי כי המודל למד את ה־manifold של תמונות אמיתיות.

הקבלה ל־PCA

PCA הוא בעצם autoencoder לינארי:

  • encoder = הטלה על הוקטורים העצמיים (כפל במטריצה $U^T$).
  • decoder = הטלה הפוכה (כפל ב־$U$).
  • אבל autoencoder עם פונקציות אקטיבציה לא־לינאריות יכול ללמוד manifolds מורכבים יותר.
נקודה למבחן: PCA הוא הגרסה הלינארית של autoencoder. אם הקשרים בנתונים לינאריים — PCA מספיק. אם לא — autoencoder עם non-linearity יעבוד טוב יותר.

שימוש: זיהוי תמונות חריגות (Anomaly Detection)

🔍תהליך
  1. אמן autoencoder על תמונות תקינות (למשל, רק ספרות 0-9 ברורות).
  2. לכל תמונה חדשה: העבר אותה דרך ה־autoencoder, חשב reconstruction loss (MSE).
  3. תמונה רגילה ← loss נמוך (האוטוקודר ראה דברים דומים).
  4. תמונה חריגה (אות, סמל, תמונה שונה לחלוטין) ← loss גבוה ← התראה.

יישומים בעולם האמיתי:

  • זיהוי פגמים בייצור (חתיכות תקינות מול פגומות).
  • זיהוי גידולים בתמונות רפואיות.
  • זיהוי תוכן חריג ברשתות חברתיות.

Drift Detection בתמונות

📘איך תמונות ״סוטות״ עם הזמן

מודל שאומן על תמונות מ־2020 יכול להיכשל ב־2026. למשל:

  • Novel topic — מסכות פנים הופיעו בתמונות אחרי 2020. מודל ישן לזיהוי פנים יכשל.
  • Diminishing topic — תמונות של מטבע מסוג ישן שלא בשימוש פוחתות.
  • Change in volume — אותם אובייקטים, אבל היחס משתנה (יותר תמונות סלולר, פחות תמונות מצלמה).

שיטה מעשית לזיהוי

משתמשים ב־autoencoder שאומן על data ישן:

  1. מודדים את ה־reconstruction loss הממוצע על data חדש שמגיע כל יום.
  2. אם ה־loss עולה באופן עקבי לעומת הבייסליין — סימן ל־drift.
  3. הנקודה שבה ה־loss מתחיל לעלות = drift point.
תזכורת: drift תופעה כללית בלמידה בעולם האמיתי, לא רק בתמונות. אבל בתמונות זה במיוחד נפוץ כי העולם הפיזי משתנה ויזואלית כל הזמן.

טעויות נפוצות

⚠חמש טעויות שמורידות נקודות
  1. Fit PCA על ה־test set. זאת דליפת נתונים (data leakage). PCA חייב להיות fit רק על ה־train. ה־test רק transform.
  2. שכחה של scaling לפני PCA. פיקסלים בטווח 0-255, אבל הם לא דורשים scaling בין משתנים (כולם באותו טווח). אם עוברים ל־data אחר עם משתנים שונים, scaling קריטי.
  3. חשיבה ש־autoencoder תמיד טוב יותר מ־PCA לתמונות. רק עם data מורכב או non-linear. ל־data לינארי, PCA פשוט ויעיל יותר.
  4. בלבול בין מספר רכיבים לאחוז שונות. ״k=2 PCA״ ≠ ״2% שונות״! עם 2 רכיבים אפשר לכסות 30-40% של ה־data, תלוי בנתונים.
  5. הנחה ש־accuracy לא ייפגע מהורדת מימד. כן ייפגע, בדרך כלל. השאלה היא כמה ויש לוודא שזה מקובל לפני שמתחייבים.

תרגילי הבנה

📝תרגיל 1 · PCA על תמונות digits
פירוש תוצאות

בנתוני digits (1797 תמונות בגודל 8×8, 10 ספרות), קיבלנו:

  • Accuracy לפני PCA (64 features): 0.978
  • Accuracy עם 31 רכיבי PCA (90% שונות): 0.949
  • Accuracy עם 2 רכיבי PCA: 0.511

סמנו נכון/לא נכון:

  1. 2 רכיבים מספיקים לחזות נכון כ־51% מהתמונות — בערך פי 5 מניחוש אקראי בין 10 מחלקות.
  2. הירידה מ־0.978 ל־0.949 (אחרי PCA ל־31) קטנה אבל מובהקת — מאבדים מידע.
  3. אם נשתמש ב־64 רכיבי PCA (כל המימדים), ה־accuracy יישאר 0.978.
  4. נסתי PCA אחרת — fit_transform גם על test — היא תיתן accuracy גבוה יותר.
  5. הירידה ב־accuracy מצביעה שהפיקסלים לא מתואמים זה עם זה.
  • 1. 51% הרבה יותר מ־1/10. נכון. אקראי על 10 מחלקות נותן 10% accuracy. 51% הרבה יותר טוב, אבל בכל זאת רע יחסית. 2 רכיבים מאבדים יותר מדי מידע.
  • 2. ירידה מובהקת אבל קטנה. נכון. ירידה של ~3% היא משמעותית סטטיסטית על 450 תצפיות, אבל לא דרמטית. הוודאי שיש איזון בין דחיסה לדיוק.
  • 3. 64 רכיבי PCA = אותו ביצוע. נכון מתמטית. PCA עם כל המימדים זה פשוט סיבוב של מערכת הצירים — אין איבוד מידע. הרגרסיה הלוגיסטית תיתן בדיוק את אותה תוצאה.
  • 4. fit_transform על test = יותר טוב. לא נכון! זה data leakage. הביצועים יכולים להיראות טובים יותר בבדיקה הזאת, אבל זה לא משקף ביצועים אמיתיים על data חדש. חוק זהב: fit רק על train.
  • 5. הירידה מצביעה על חוסר תאמת. לא נכון, ההפך. העובדה ש־31 רכיבים מסבירים 90% מהשונות מצביעה שיש תאמת חזקה — אחרת היינו צריכים את כל 64 הרכיבים.
נכונות: 1, 2, 3.
📝תרגיל 2 · בחירה בין PCA ל־Autoencoder
תרחישים מעשיים

לכל תרחיש, מה עדיף — PCA או Autoencoder? נמק.

  1. יש 100 תמונות, וכל תמונה היא 8×8 = 64 פיקסלים. רוצים לויזואלזציה ב־2D.
  2. יש מיליוני תמונות מורכבות 256×256 ורוצים לזהות אנומליות.
  3. הנתונים לינאריים — features מתואמים זה עם זה ביחס לינאריים.
  4. רוצים הסבר מה כל component מייצג (interpretability).
תרחיש 1 · 100 תמונות, ויזואליזציה 2D

PCA. מהיר, פשוט, מאפשר ויזואליזציה. גם — אין מספיק נתונים לאמן autoencoder.

תרחיש 2 · מיליוני תמונות מורכבות, anomaly detection

Autoencoder. ה־scale תומך באימון, ו־anomaly detection הוא הקייס שלו הקלאסי. תמונות 256×256 מורכבות יותר מ־8×8 ויהיו תבניות לא־לינאריות.

תרחיש 3 · נתונים לינאריים

PCA. כשהקשרים לינאריים, PCA אופטימלי. autoencoder לינארי יתן בדיוק את אותה תוצאה אבל יהיה איטי יותר לאימון.

תרחיש 4 · interpretability

PCA. אפשר לקרוא את ה־loadings של כל PC ולהסיק מה הוא ״לוכד״. ב־autoencoder עם 5 שכבות וטרנספורמציות לא־לינאריות — לאיש אין מושג מה ה־bottleneck באמת מייצג.

PCA בתרחישים 1, 3, 4. Autoencoder בתרחיש 2.
📝תרגיל 3 · Drift detection
תרחיש drift

אתם בונים מערכת לזיהוי פנים אוטומטית בכניסה למשרד. המערכת אומנה ב־2018. ב־2020 הופיע covid והעובדים מתחילים לחבוש מסכות. סמנו נכון/לא נכון:

  1. זאת דוגמה ל־״novel topic drift״ — קונספט חדש (מסכה) הופיע ב־data.
  2. אם נריץ autoencoder שאומן על תמונות ללא מסכות, ה־reconstruction loss על תמונות עם מסכות יהיה גבוה.
  3. הפתרון הנכון הוא להמשיך להשתמש במודל הקיים, כי הוא עובד טוב על data ישן.
  4. אם רק 5% מהעובדים חובשים מסכה — אין drift, כי זה אחוז נמוך.
  • 1. Novel topic drift. נכון. ״מסכה״ הוא נושא ויזואלי חדש שלא היה ב־data ההיסטורי.
  • 2. Reconstruction loss גבוה. נכון. ה־autoencoder לא ראה מסכות באימון — הוא ייכשל בשחזורן ← loss גבוה. זאת הדרך לזהות drift אוטומטית.
  • 3. להמשיך עם המודל הישן. לא נכון. אם יש drift משמעותי, חייבים לאמן מחדש או לכל הפחות לעדכן עם תמונות עם מסכות. אחרת המערכת תיכשל לזהות עובדים.
  • 4. 5% = אין drift. לא נכון. drift הוא שינוי בהתפלגות, לא תלוי באחוז מוחלט. גם 5% הם שינוי — וכנראה יגדל. צריך לעקוב.
נכונות: 1, 2.  ·  לא נכונות: 3, 4.

סיכום מהיר לזיכרון

📌תשע נקודות שאתה חייב לזכור
  1. תמונה היא וקטור — תמונה $8 \times 8$ היא וקטור באורך 64, ואותם כלי ML עובדים עליה.
  2. Digits dataset — 1797 תמונות, 64 פיקסלים, 10 מחלקות.
  3. Logistic regression על raw pixels נותן 97.78% accuracy  ·  PCA עם 31 רכיבים נותן 94.89%.
  4. 31 רכיבים מספיקים ל־90% מהשונות, 40 רכיבים ל־95% — הוכחה ל־redundancy בפיקסלים.
  5. חוק זהב — fit PCA על train בלבד, transform על test, אחרת data leakage.
  6. PCA הוא autoencoder לינארי — Autoencoders עם non-linearity יכולים ללמוד מורכבות יותר.
  7. Anomaly detection עם autoencoder — train על תקין, גילוי לפי reconstruction loss.
